#Intro Responsive and useDeferredValue()

useDeferredValue is a React Hook that lets you delay updating a value so React can keep the UI responsive, especially during expensive renders.

useDeferredValue = "Use the old value until React has time to update the new one"

Think of it like this:

- User types → input updates immediately
- Expensive component updates → delayed slightly
- Result → smooth typing, no lag

React shows the old value until the new one is ready.

```tsx
startTransition(() => {
  setValue(event.currentTarget.value);
});
```

```tsx
const deferredValue = useDeferredValue(value);
const isPending = deferredValue !== value;
```

When part of the UI updates based on user input, the input itself should update immediately as the user types. However, the results that depend on that input (such as a list of matching ships) can update more slowly.

If you wrap the input update in a transition, the input value may not update immediately, which creates a poor user experience. Instead, the input should update synchronously, while the results update in a transition.

This is where useDeferredValue helps. It lets you defer updating the slower part of the UI while keeping the input responsive. It returns a deferred version of a value from props or state.

React will render the component twice: once with the current value and once with the deferred (previous) value. If the new value isn’t ready yet (for example, because it is suspended), React will continue showing the previous value until the new one is ready. This allows you to show pending states while keeping the UI responsive.

## Example without useDeferredValue (can lag)

Type → expensive render → input freezes

```tsx
function Search() {
  const [query, setQuery] = useState('');

  const results = expensiveFilter(query);

  return (
    <>
      <input value={query} onChange={(e) => setQuery(e.target.value)} />
      <Results list={results} />
    </>
  );
}
```

## Example with useDeferredValue (smooth)

Type → input updates immediately
→ expensive render happens later

```tsx
import { useState, useDeferredValue } from 'react';

function Search() {
  const [query, setQuery] = useState('');

  const deferredQuery = useDeferredValue(query);

  const results = expensiveFilter(deferredQuery);

  return (
    <>
      <input value={query} onChange={(e) => setQuery(e.target.value)} />
      <Results list={results} />
    </>
  );
}
```

- Input updates immediately
- Filtering waits
- UI stays responsive

It does NOT debounce

### Debounce:

- waits fixed time (e.g., 300ms)

### Deferred value:

- lets React decide when to update
- based on rendering priority

## Advanced example (realistic)

```tsx
function App() {
  const [text, setText] = useState('');
  const deferredText = useDeferredValue(text);

  return (
    <>
      <input value={text} onChange={(e) => setText(e.target.value)} />

      <SlowList text={deferredText} />
    </>
  );
}
```

When you SHOULD use it / Use when:

- typing causes slow rendering
- large lists
- expensive calculations
- UI stutters

When NOT needed / Don’t use when:

- UI is already fast
- small components
- simple forms

Summary useDeferredValue:

- delays updating heavy UI parts
- keeps input responsive
- improves performance
- used in concurrent rendering
